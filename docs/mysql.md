# MySQL: физический бэкап

Физический бэкап для MySQL в текущей версии агента **не поддерживается**.

Доступен только логический режим (`mysqldump`). Он покрывает большинство сценариев: полный экспорт схемы и данных, перенос базы, восстановление на другой сервер.

## Почему физический режим не реализован

В отличие от PostgreSQL (который поставляется с утилитой `pg_basebackup`) и MSSQL (нативная команда `BACKUP DATABASE`), в Community Edition MySQL нет встроенного инструмента для физического бэкапа. Единственная рабочая альтернатива — Percona XtraBackup, сторонняя утилита с нетривиальными требованиями:

- XtraBackup устанавливается отдельно и должен быть совместим с конкретной мажорной версией MySQL.
- Для физического бэкапа агент должен находиться на том же хосте, что и MySQL, и иметь прямой доступ к его рабочей директории (`/var/lib/mysql`).
- Восстановление требует промежуточного шага подготовки бэкапа (`xtrabackup --prepare`) перед тем, как данные можно использовать.

Это делает поддержку физического режима нецелесообразной для большинства пользователей.

## Поиск бинарников `mysqldump` / `mysql`

Агент вызывает утилиты MySQL-клиента (`mysqldump` для бэкапа, `mysql` для восстановления) напрямую. Порядок резолва каталога с бинарниками:

1. Если в `ConnectionConfig.BinPath` указан путь — используется он.
2. Иначе агент ищет установку на хосте:
   - Windows: `C:\Program Files\MySQL\MySQL Server *\bin` и `C:\Program Files (x86)\MySQL\MySQL Server *\bin`. Если найдено несколько — выбирается каталог с наибольшей версией.
   - Linux: `/usr/local/mysql/bin` (установка из tarball).
3. Если каталог не найден — fallback на `PATH`.

В отличие от PostgreSQL, агент не опрашивает сервер о версии: MySQL-клиент обратно совместим, и `mysqldump` подходящей или более свежей версии работает корректно.

**Когда задавать `BinPath` явно.** На Windows инсталлятор MySQL часто добавляет `bin` только в `User PATH`, но агент работает как служба (LocalSystem / отдельный аккаунт) и этот PATH не видит. Либо авто-резолв находит не ту установку. В обоих случаях задайте каталог явно:

```json
{
  "Connections": [
    {
      "Name": "mysql-main",
      "DatabaseType": "Mysql",
      "Host": "127.0.0.1",
      "Port": 3306,
      "Username": "backup_user",
      "Password": "...",
      "BinPath": "C:\\Program Files\\MySQL\\MySQL Server 8.0\\bin"
    }
  ]
}
```

Override бьёт всё остальное — ни скан стандартных каталогов, ни `PATH` не опрашиваются.
